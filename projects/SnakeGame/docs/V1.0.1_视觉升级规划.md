# V1.0.1 视觉升级规划

**版本**: V1.0.1  
**类型**: 视觉体验优化  
**参考风格**: 微信跳一跳（简约、柔和色彩、圆润造型）  
**视角**: 2D 俯视（保持经典玩法）  
**创建日期**: 2026-03-21  

---

## 一、现状分析

### 1.1 当前问题

| 问题 | 描述 | 优先级 |
|------|------|--------|
| 界面单调 | 纯色背景，缺乏层次感 | P0 |
| 蛇身简陋 | 简单方块拼接，无辨识度 | P0 |
| 食物无吸引力 | 缺乏视觉反馈 | P1 |
| 动画生硬 | 移动、转向无过渡效果 | P1 |
| UI 缺失 | 无开始界面、结算界面 | P0 |
| 特效空白 | 无粒子、无光效 | P2 |

### 1.2 设计原则

✅ **保持 2D 俯视视角** - 经典玩法，方向判断清晰  
✅ **简约不简单** - 减少视觉干扰，但不牺牲美感  
✅ **柔和亲和** - 圆润造型、低饱和度色彩  
✅ **流畅反馈** - 60FPS 动画、即时响应

### 2.3 造型规范

#### 蛇（2D 俯视）

| 部位 | 造型 | 尺寸 | 说明 |
|------|------|------|------|
| 头部 | 圆形 + 眼睛 | 32x32px | 白色眼睛 + 黑色瞳孔 |
| 身体 | 圆角矩形 | 28x28px | 圆角半径 8px |
| 尾巴 | 渐细三角形 | 24x24px | 平滑过渡 |

**蛇身连接**: 节段之间重叠 4px，避免缝隙

#### 食物

- **基础**: 圆形（直径 24px）
- **高光**: 左上角 1/4 圆弧白色高光（透明度 70%）
- **叶梗**: 顶部 2 像素绿色小椭圆
- **阴影**: 底部轻微椭圆阴影（透明度 20%）

#### 地面网格

- **线条**: 1px 圆角线
- **颜色**: `#B8E6F5` 透明度 40%
- **间距**: 32px（与蛇身节段对齐）
- **样式**: 虚线或实线（待视觉测试）

## 四、技术实现方案

### 4.1 渲染架构

```
┌─────────────────────────────────────────┐
│           Game (主循环)                  │
├─────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────────┐   │
│  │ WorldLayer  │  │    UILayer      │   │
│  │ (游戏世界)   │  │   (UI 界面)      │   │
│  ├─────────────┤  ├─────────────────┤   │
│  │ Background  │  │  StartScreen    │   │
│  │ Grid        │  │  PauseScreen    │   │
│  │ Food        │  │  GameOverScreen │   │
│  │ Snake       │  │  HUD(分数)      │   │
│  │ Particles   │  │                 │   │
│  └─────────────┘  └─────────────────┘   │
└─────────────────────────────────────────┘
```

### 4.2 图层顺序（从后到前）

```csharp
1. BackgroundLayer    // 渐变背景
2. GridLayer          // 地面网格
3. FoodLayer          // 食物（带浮动动画）
4. SnakeLayer         // 蛇（头→身→尾）
5. ParticleLayer      // 粒子特效
6. HUDLayer           // 分数显示
7. UILayer            // 界面（开始/暂停/结算）
```

### 4.4 核心代码示例

```csharp
// 蛇身渲染（圆角矩形 + 渐变）
public class SnakeSegment : IRenderable
{
    private Color _headColor = new Color(81, 216, 166);
    private Color _bodyColor = new Color(123, 230, 194);
    
    public void Draw(SpriteBatch spriteBatch, Vector2 position)
    {
        // 绘制圆角矩形（使用纹理或程序生成）
        spriteBatch.Draw(_roundRectTexture, position, null, _bodyColor);
        
        // 边缘描边
        spriteBatch.Draw(_roundRectOutlineTexture, position, _edgeColor);
    }
}

// 食物浮动动画
public class Food : IRenderable
{
    private float _floatOffset = 0;
    private const float FloatSpeed = 2.0f;
    private const float FloatAmplitude = 3.0f;
    
    public void Update(GameTime gameTime)
    {
        _floatOffset = (float)Math.Sin(gameTime.TotalGameTime.TotalSeconds * FloatSpeed) 
                       * FloatAmplitude;
    }
    
    public void Draw(SpriteBatch spriteBatch, Vector2 position)
    {
        Vector2 drawPos = position + new Vector2(0, _floatOffset);
        spriteBatch.Draw(_foodTexture, drawPos, Color.White);
    }
}
```

### 4.5 性能优化

| 优化项 | 方案 | 预期收益 |
|--------|------|----------|
| SpriteBatch 合并 | 同图层批量绘制 | 减少 Draw 调用 80% |
| 纹理图集 | 合并小图为大图 | 减少纹理切换 |
| 对象池 | 粒子复用 | 减少 GC 压力 |
| 脏矩形渲染 | 只重绘变化区域 | 减少 GPU 负载 |

## 七、风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| 资源制作延期 | 高 | 中 | 使用临时占位图（程序绘制圆角矩形），先开发后替换 |
