## 1. 概述

本文档定义 SnakeGame 应用层的异常分类体系，包括：
- 异常严重级别定义
- 异常处理策略
- 用户提示规范
- 日志记录规范

---

## 2. 异常严重级别

| 级别 | 名称 | 说明 | 应用行为 |
|------|------|------|---------|
| `Critical` | 致命错误 | 应用无法继续运行 | 记录日志 + 显示错误 + 退出 |
| `Recoverable` | 可恢复错误 | 功能受限但可继续 | 记录日志 + 显示提示 + 继续 |
| `User` | 用户错误 | 不影响应用运行 | 仅显示提示 |

### 2.1 Critical（致命错误）

**触发场景**：
- `OutOfMemoryException` - 内存不足
- 图形设备初始化失败
- 不可恢复的系统资源错误

**处理策略**：
```csharp
// 记录致命错误
logger.Critical("图形设备初始化失败", exception);

// 显示用户友好提示
ShowMessage("内存不足，无法继续运行。\n请关闭其他应用程序后重试。");

// 退出应用
Exit(1);
```

### 2.2 Recoverable（可恢复错误）

**触发场景**：
- `FileNotFoundException` - 文件不存在
- `JsonException` - 配置文件格式错误
- `IOException` - 文件读写错误
- `UnauthorizedAccessException` - 权限不足

**处理策略**：
```csharp
// 记录警告日志
logger.Warning("配置文件损坏，使用默认值", exception);

// 显示提示（可选）
ShowToast("配置文件损坏，已恢复默认设置");

// 继续运行，使用默认值
LoadDefaultSettings();
```

### 2.3 User（用户错误）

**触发场景**：
- `ArgumentException` - 无效参数
- `InvalidOperationException` - 状态不允许的操作

**处理策略**：
```csharp
// 不记录日志（用户操作错误无需记录）
// 仅显示提示
ShowToast("当前状态无法执行此操作");
```

## 3. 异常分类映射

### 3.1 系统异常

| 异常类型 | 级别 | 用户提示 | 恢复提示 |
|---------|------|---------|---------|
| `OutOfMemoryException` | Critical | 内存不足，无法继续运行 | 请关闭其他应用程序后重试 |
| `UnauthorizedAccessException` (graphics) | Critical | 无法访问图形设备 | 请检查显卡驱动是否正常 |
| `UnauthorizedAccessException` | Recoverable | 无法访问文件 | 请检查文件权限 |

### 3.2 IO 异常

| 异常类型 | 级别 | 用户提示 | 恢复提示 |
|---------|------|---------|---------|
| `FileNotFoundException` | Recoverable | 文件不存在，将使用默认设置 | - |
| `DirectoryNotFoundException` | Recoverable | 目录不存在，将自动创建 | - |
| `IOException` | Recoverable | 文件读写错误 | 显示具体错误信息 |
| `JsonException` | Recoverable | 数据格式错误，将使用默认值 | - |

### 3.3 业务异常

| 异常类型 | 级别 | 用户提示 | 恢复提示 |
|---------|------|---------|---------|
| `ArgumentException` | User | 无效的操作 | - |
| `InvalidOperationException` | User | 当前状态无法执行此操作 | - |

### 3.4 未知异常

| 场景 | 级别 | 用户提示 | 恢复提示 |
|------|------|---------|---------|
| 未匹配的异常 | Recoverable | 发生未知错误 | 显示异常类型名称 |

## 4. 日志记录规范

### 4.1 日志级别

| 日志级别 | 使用场景 | 是否记录异常堆栈 |
|---------|---------|-----------------|
| `Debug` | 调试信息 | 否 |
| `Info` | 正常操作信息 | 否 |
| `Warning` | 可恢复错误 | 是 |
| `Error` | 严重错误 | 是 |
| `Critical` | 致命错误 | 是 |

### 4.2 日志格式

```
[2026-03-23 00:35:00.123] [WARNING] 排行榜文件损坏，使用空列表
  Exception: System.IO.FileNotFoundException
  Message: Could not find file 'leaderboard.json'
  StackTrace: at SnakeGame.App.LeaderboardStore.Load()...
```

### 4.3 日志存储

- **位置**：`%APPDATA%/SnakeGame/logs/game.log`（Windows）/ `~/.config/SnakeGame/logs/game.log`（Linux）
- **滚动策略**：按日期归档，格式 `game_yyyyMMdd.log`
- **清理策略**：保留最近 30 天

## 5. 代码示例

### 5.1 使用异常分类器

```csharp
try
{
    LoadLeaderboard();
}
catch (Exception ex)
{
    var classification = ExceptionClassifier.Classify(ex);

    if (ExceptionClassifier.ShouldLog(classification.Severity))
    {
        logger.Warning(classification.UserMessage, ex);
    }

    if (ExceptionClassifier.ShouldShowToUser(classification.Severity))
    {
        ShowMessage(classification.UserMessage, classification.RecoveryHint);
    }

    if (!ExceptionClassifier.CanContinue(classification.Severity))
    {
        Exit(1);
    }
}
```

### 5.2 文件操作最佳实践

```csharp
public LeaderboardData Load()
{
    try
    {
        var json = File.ReadAllText(_filePath);
        return JsonSerializer.Deserialize<LeaderboardData>(json) ?? new LeaderboardData();
    }
    catch (FileNotFoundException)
    {
        // 文件不存在，返回默认数据
        _logger.Info("排行榜文件不存在，创建新文件");
        return new LeaderboardData();
    }
    catch (JsonException ex)
    {
        // JSON 格式错误，记录日志并返回默认数据
        _logger.Warning("排行榜文件格式错误，将重置", ex);
        return new LeaderboardData();
    }
}
```
