# SnakeGame V1.0 发版声明

## 1. 版本信息

- 产品名称：`SnakeGame`
- 版本号：`V1.0`
- 发布日期：`2026-03-20`
- 版本定位：首个可运行、可编译、可交付的中文桌面版贪吃蛇实验版本
- 技术路线：`C# + .NET 8 + MonoGame`

## 2. 版本概述

`V1.0` 版本完成了项目从需求规划、核心规则建模、单元测试设计到桌面应用落地的闭环，实现了多关卡贪吃蛇的基础可玩版本。

本版本以“核心玩法可运行、核心规则可测试、项目结构可持续演进”为目标，优先保证：

- 核心规则闭环完整
- 中文界面与中文文档齐备
- 基础交互流程可用
- `SnakeGame.Core` 具备高覆盖率单元测试保护

## 3. 产品主要功能设计

### 3.1 游戏模式

- **闯关模式**：从第 1 关开始逐关推进
- **练习模式**：开始界面可直接选关进入练习

### 3.2 关卡与成长机制

- 游戏采用多关卡设计
- 每关通过 `目标长度` 作为通关条件，而不是依赖“苹果必须覆盖全部可用空间”
- 通关后自动进入下一关
- 关卡提升后难度递增，主要体现在：
  - 基础移动速度提升
  - 固定障碍数量增加
  - 高等级关卡启用移动障碍

### 3.3 苹果生成策略

- 苹果不会简单随机刷新到任意空格
- 生成时会执行可达性与安全性校验
- 仅当候选位置满足以下条件时才允许生成：
  - 蛇头存在路径可以到达该苹果
  - 吃到苹果后，蛇仍存在继续生存的安全路径
- 运行时生成采用合法候选点随机选择，避免苹果长期固定刷在同一位置

### 3.4 障碍系统

- **固定障碍**：阻挡蛇移动，碰撞即失败
- **移动障碍**：
  - 高关卡启用
  - 沿预设轨道移动
  - 主动撞到蛇头会导致游戏结束
  - 与蛇身被动接触时会暂停
  - 脱离蛇身后恢复原方向移动
  - 遇到轨道端点或固定障碍时会反向

### 3.5 操作与交互

- 支持 `方向键` 和 `WASD`
- 支持 `P` 暂停/继续
- 支持 `Esc` 返回主菜单
- 支持双击并按住当前前进方向触发加速：
  - 双击判定窗口：`200ms`
  - 加速倍率：`x2`
  - 松开方向键后恢复为当前关卡基础速度

### 3.6 数据与设置

- 本地排行榜持久化
- 本地音频设置持久化
- 启动异常自动写入崩溃日志

### 4.1 `SnakeGame.Core`

负责纯业务逻辑，不依赖 MonoGame：

- 游戏领域模型与状态快照
- 苹果生成与路径安全校验
- 关卡容量校验
- 通关规则判断
- 移动障碍解析
- 双击加速判定
- 分数策略与音频设置服务
- `GameEngine` 主状态推进

### 4.2 `SnakeGame.App`

负责桌面应用层：

- MonoGame 启动与主循环
- 菜单、选关、暂停、结果页
- 棋盘/HUD 绘制
- 输入接线
- 本地排行榜文件存储
- 本地配置文件存储
- 中文字体与内容管线
