# 基于测试驱动开发的中文版多关卡贪吃蛇计划

## 概述

- 使用 MonoGame 开发 Windows 桌面版 C# 贪吃蛇，界面、文案、配置说明和后续文档全部使用中文。
- 游戏分为 `闯关模式` 和 `选关练习模式`：闯关模式从第 1 关开始并写入总排行榜；选关练习允许指定起始关卡，但不写入总排行榜，避免和选关机制冲突。
- 关卡核心由“目标长度通关”驱动，不再依赖填满地图；难度随关卡提升体现在基础速度变快、固定障碍增多、第 5 关起加入移动障碍。
- 采用测试驱动开发：`Game.Core` 负责纯规则与状态机，`Game.App` 负责 MonoGame 输入、渲染、音频与中文界面绑定；先写核心测试，再逐步补实现。

## 核心接口与规则

- `LevelConfig`：定义关卡编号、目标长度、初始蛇长、基础步进间隔、固定障碍数量、是否启用移动障碍、加速倍数、移动障碍轨道。
- `GameMode`：`Story`（闯关，记总榜）与 `Practice`（练习，不记总榜）。
- `MovingObstacle`：包含轨道、当前方向、状态 `Moving/PausedByBody`、恢复方向；每条轨道仅允许一个移动障碍。
- `AudioSettings`：至少包含 `BgmEnabled`、`SfxEnabled`、`BgmVolume`、`SfxVolume`，本地文件持久化。
- `GameState`：至少覆盖主菜单、关卡选择、游戏中、暂停、过关结算、游戏结束。

## 实现要点

### 关卡生成与通关

- 每关开始前先验证地图可玩性，确保“安全可用格数上限”大于目标长度并留出冗余；非法布局直接重生成，不允许在游戏中临时删障碍改规则。
- 达到 `TargetLength` 立即过关，重置蛇身并加载下一关配置；第 1 关基础速度刻意放慢。

### 苹果生成与安全校验

- 苹果候选格必须排除墙、固定障碍、蛇身、移动障碍当前位置，以及移动障碍专用轨道格。
- 对每个候选格执行路径验证：先找出蛇头到苹果的可行路径，再做“虚拟吃取模拟”，按真实规则更新虚拟蛇身，最后验证吃到后的新蛇头仍可到达新蛇尾；只有通过强校验的格子才能生成苹果。
- 若扫描完整个候选集合仍无合法点，则判定当前局面为死局并结束本局，不在局内修改障碍。

### 移动障碍规则

- 第 5 关起启用沿固定轨道往返的移动障碍。
- 移动障碍若下一步将进入蛇头当前格、蛇头下一步目标格，或与蛇头发生交换位置，立即 `GameOver`。
- 若下一步目标格被蛇身占用，则障碍停在原位并进入 `PausedByBody`；当该目标格重新空出后，沿原方向继续移动。
- 撞到轨道端点或固定障碍时反向；不允许与蛇身或其他障碍重叠显示。

### 输入与加速

- 支持方向键和 WASD。
- 仅当玩家对“当前运动方向”在可配置时间窗内快速双击，且第二次保持按住，才触发加速；默认时间窗 200ms、倍率 2x。
- 松开该键或方向改变立即恢复当前关卡基础速度；按“按下沿”判定双击，避免长按误触。

### 界面与中文化

- 主菜单提供“开始闯关”“选关练习”“排行榜”“设置”“退出”。
- 选关界面展示关卡中文简介、目标长度、速度级别、障碍说明。
- HUD 显示当前关卡、分数、当前长度、目标长度、暂停提示、BGM 状态。
- 必须配置可正常显示中文的字体资源与 `SpriteFont` 字形集，避免汉字缺失。

### 排行榜与分数

- 本地排行榜默认只记录闯关模式成绩；练习模式仅显示本次结果，不进入总榜。
- 总榜至少记录分数、到达关卡、时间和模式，保存为本地 JSON。

### 音频

- 接入 BGM 与音效（转向、吃苹果、碰撞、过关/失败；移动音效可选且需节流）。
- 设置界面支持 BGM 开关，同时提供 SFX 开关或音量；配置重启后保持。

## 测试计划

- 验证每次苹果生成都满足“可到达且吃后蛇头仍可到尾”的强校验；在高障碍和长蛇场景下不生成伪安全苹果。
- 验证关卡开始前的布局校验有效，目标长度不会超过关卡可实现上限；闯关时达到目标长度稳定切换下一关。
- 验证第 5 关后的移动障碍状态机：蛇头主动碰撞必死、蛇身阻挡仅暂停、脱离后恢复、撞端点反向、交换位置判定为碰撞。
- 验证双击加速只对当前运动方向生效，长按不误触，松开即恢复，基础速度与加速速度能随关卡正确变化。
- 验证主菜单、选关、暂停、结算、排行榜、设置的中文显示完整；中文字体无乱码或缺字。
- 验证闯关模式写总榜、练习模式不写总榜；BGM/SFX 设置可保存并在下次启动时生效。

## 默认假设

- 默认技术栈为 `MonoGame + .NET + xUnit`；如果后续改 Unity，也仍保留“纯规则核心 + 引擎壳层”的测试驱动结构。
- TDD 的重点是玩法规则与状态机，不强求渲染和音频设备层百分百自动化。
- 所有中文文案、字体资源、音频资源均由本地工程提供；排行榜和设置为本地文件持久化。
