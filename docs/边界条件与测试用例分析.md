# 边界条件与测试用例分析

## 分析目标

- 把执行计划中的玩法约束拆成可验证的单元测试。
- 优先覆盖“最容易出现逻辑漏洞”的区域：苹果安全性、移动障碍同步结算、加速判定、关卡可玩性、模式与排行榜策略。

## 边界条件分类

### 苹果生成

- 地图仅剩一个合法苹果点时，必须稳定找到该点。
- 候选格可到达，但吃完后无法再接回蛇尾时，必须判为非法。
- 移动障碍轨道经过的格子不允许生成苹果，即使该格当前为空。
- 地图不存在合法苹果点时，必须返回死局而不是落到非法格。

### 关卡与布局

- 目标长度超过安全容量时，关卡布局必须判为不可玩。
- 初始蛇长大于等于目标长度时，关卡配置非法。
- 第 1 关基础步进应慢于后续关卡，用例需验证节奏设定不会反转。

### 移动障碍

- 轨道到端点时必须反向。
- 障碍下一步撞蛇头当前格、蛇头下一步目标格、或与蛇头交换位置时都要判死。
- 障碍下一步进入蛇身格时，只暂停不致死，且蛇身移开后恢复。
- 多个障碍同时存在时，每个障碍都应独立处理自己的轨道与状态。

### 输入与加速

- 只有对“当前运动方向”的快速双击才触发加速。
- 第二次必须保持按住；松开后立即恢复正常速度。
- 超过双击时间窗、按错方向、或先变向再双击，都不应触发加速。

### 排行榜与设置

- 闯关模式成绩写入总榜，练习模式不写。
- 音频设置文件不存在时，必须加载默认值。
- 切换 BGM 或 SFX 后必须持久化，供下次启动读取。

## 建议的测试层次

### 单元测试

- 直接测试 `AppleSpawnService`、`LevelLayoutValidator`、`LevelProgressionService`、`MovingObstacleResolver`、`AccelerationResolver`、`ScoreSubmissionPolicy`、`AudioSettingsService`。
- 不依赖 MonoGame，不创建窗口、不加载字体、不播放真实音频。

### 集成测试

- 后续补 `GameEngine.Step` 后，增加固定输入序列的多步状态推进测试。
- 重点覆盖“吃苹果 -> 增长 -> 过关 -> 切换关卡”以及“移动障碍参与碰撞”的完整流程。

### 手工验收

- 中文字体是否完整显示。
- 加速手感是否自然。
- BGM 和音效是否存在过度打断或重复播放问题。

## 结论

- 当前 TDD 范围应优先锁定核心规则，不把 MonoGame 视图层纳入单元测试。
- 当新增玩法规则时，必须先补这份分析文档，再补对应测试，再进入实现。
