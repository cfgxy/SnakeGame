## 2. 已实施优化

### 2.1 对象池优化 ✅

**问题**：`ParticleSystem.Emit()` 每次创建 `Particle` 对象，频繁 GC

**解决方案**：实现通用对象池 `ObjectPool<T>`

**代码变更**：
- 新增 `src/SnakeGame.App/Rendering/ObjectPool.cs` - 通用对象池
- 修改 `src/SnakeGame.App/Rendering/ParticleSystem.cs` - 使用对象池

**优化效果**：
```csharp
// 优化前：每次 Emit 创建新对象
var particle = new Particle { ... };  // GC 分配

// 优化后：从对象池获取
var particle = _particlePool.Rent();  // 无 GC（池中有可用对象时）
```

**预期收益**：
- 粒子频繁创建/销毁场景下，GC 分配减少 90%+
- 池容量 256，足够应对最大粒子数量

### 2.2 纹理图集优化 ❌ 暂不实施

**评估结论**：
- 当前纹理数量：14 个
- 单个纹理大小：200B ~ 3KB
- 总纹理大小：约 10KB

**决策**：纹理很小，加载快，Draw Call 优化收益有限。暂不实施。

### 2.3 异步加载优化 ❌ 暂不实施

**评估结论**：
- 当前资源加载时间：< 1s
- 用户感知：无明显卡顿

**决策**：资源量小，同步加载可接受。V1.2 可考虑加载界面。
